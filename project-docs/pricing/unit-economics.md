# Юнит-экономика v2 — исследование и пересчёт

> Пересборка с нуля на основе прайсов поставщиков (май 2026) и карты действий из кода.  
> **Заменяет** модель от 2026-05-30 (таблицы с маржой 88–90% **завышали** прибыльность — см. § «Расхождения»).  
> Курс для пересчёта: **95 ₽ / USD**. Источники — в §1.

## 1. Исследование: реальные цены поставщиков

### 1.1 TikHub

**Модель:** pay-as-you-go, без абонплаты; списание только при HTTP 200.  
**Скидки:** 0% до 1 000 req/день → до **50%** при 30 000+ req/день ([docs](https://docs.tikhub.io/4592751m0)).  
**Бесплатно:** ~50 запросов на тест ([tikhub.io/pricing](https://tikhub.io/pricing)).

| Endpoint (используется в Viral) | Файл | Цена / запрос | Источник |
|--------------------------------|------|--------------:|----------|
| `GET …/instagram/v2/search_reels` | `tikhubInstagram.ts` | **$0,002** | [docs](https://docs.tikhub.io/387337819e0) |
| `GET …/instagram/v2/fetch_user_reels` | `tikhubInstagram.ts` | **~$0,001–0,002** (база $0,001; v2 — уточнить в кабинете) | [docs v1 reels](https://docs.tikhub.io/387337788e0), [pricing intro](https://docs.tikhub.io/4592751m0) |
| `GET …/instagram/v3/get_post_info_by_code` | транскрибация / refresh | **~$0,001** (базовый тариф) | [pricing intro](https://docs.tikhub.io/4592751m0) |
| `GET …/instagram/v2/fetch_post_info` | fallback поста | **~$0,001** | idem |

**Как Viral тратит TikHub (из кода):**

| Действие пользователя | Вызовов TikHub | COGS (без скидки) |
|----------------------|---------------:|------------------:|
| Ingest IG при поиске | 1× search_reels | **$0,002** (~0,19 ₽) |
| Добавить IG-конкурента | до 3× fetch_user_reels (+ retry) | **$0,003–0,006** |
| Refresh Reels конкурента | до 3× fetch_user_reels | **$0,003–0,006** |
| Daily sync IG-конкурента | 1× fetch_user_reels / профиль / день | **~$0,001** |
| Транскрибация IG (нет субтитров) | 1–2× post info | **$0,001–0,002** |

**Какой тариф TikHub нужен Viral:** отдельный план не требуется — **pay-as-you-go + пополнение баланса**. При 500–2 000 req/день скидка 10–20% realistic. Enterprise ($3 000+) — только при &gt;30k req/день.

### 1.2 DeepSeek (`deepseek-v4-flash`, код: `deepseek-generate.ts`)

Официальные цены ([api-docs.deepseek.com](https://api-docs.deepseek.com/quick_start/pricing)):

| Бакет | $ / 1M токенов |
|-------|---------------:|
| Input cache miss | $0,14 |
| Input cache hit | $0,0028 |
| Output | $0,28 |

Параметры в коде: `max_tokens: 1800`, `temperature: 0.55`, **полная история чата** каждый вызов.

| Сценарий | Input tok | Output tok | COGS USD | COGS ₽ |
|----------|----------:|-----------:|---------:|-------:|
| Короткий (1-й сценарий, мало истории) | ~4 000 | ~800 | **~$0,0008** | **~0,08 ₽** |
| Типичный (профиль + 2 референса + 4 сообщения) | ~12 000 | ~1 400 | **~$0,0021** | **~0,20 ₽** |
| Длинный (10+ сообщений в чате) | ~35 000 | ~1 800 | **~$0,0054** | **~0,51 ₽** |

*Cache hit для system prompt снижает input на ~30–50% — в расчётах не заложено (оптимизм).*

## 2. Карта действий пользователя (из кода)

| Действие | Токены (код) | COGS API | COGS infra | Класс |
|----------|-------------:|----------|------------|-------|
| Первый поиск / лента | 0 | YT+TikHub **если ingest** | DB + **долгий HTTP** | 🟡→🔴 при ingest |
| «Ещё ролики» | 5 | как поиск при thin pool | DB + HTTP | 🟡→🔴 |
| Throttle 15 мин / query | — | кэш-поведение | `appRuntimeState` | 🟢 |
| Повторный поиск (throttled) | 0 | **0** внешний | DB only | 🟢 |
| Сохранить ролик | 0 | 0 | Postgres write | 🟢 |
| Открыть карточку / saved state | 0 | 0 | read | 🟢 |
| Тренды poll (120 с) | 0 | 0 | invocations + DB | 🟡 |
| Lazy refresh trends | 0 | возможен YT/TikHub | тяжёлый POST | 🔴 |
| Генерация сценария | 20 | DeepSeek | HTTP 60s | 🟡 |
| Транскрибация | 5 | Groq или 0 | HTTP | 🟢–🟡 |
| Добавить IG-конкурента | 30 | 3–6× TikHub | sync HTTP | 🔴 |
| Refresh Reels | 30 | 3–6× TikHub | sync HTTP | 🔴 |
| Daily sync / конкурент | 5 | 1× TikHub или YT | batch | 🟡 |

**Критически дорогие:** sync ingest в feed, competitor add/refresh, lazy trends refresh.  
**Бесплатные для нас (пока):** browse DB, saves, throttled re-search.  
**Скрытый налог:** trends poll **вне вкладки Home** (аудит) — лишние invocations.

| 3. Активный блогер | 25 ingest search, 30 load-more, 12 script, 8 transcribe, 3×15 sync | ~55 | ~180 | **~235** | 2 990 | **92%** |

## 8. Если запускать оплату завтра

| Решение | Рекомендация |
|---------|--------------|
| **Тарифы** | Creator **2 990 ₽** + Visioner **14 990 ₽**; добавить **Producer 6 990 ₽** в первые 60 дней если &gt;20% юзеров упираются в лимит конкурентов |
| **Токены** | Free **60** · Trial **450** · Creator **1 800/мес** · Producer **5 500** · Visioner **12 000** |
| **Trial** | **Да**, 3 дня + карта → Creator; **450 токенов** |
| **Free** | **Да**, но **без external ingest** — иначе ~0,19 ₽ × каждый любопытный |
| **Средний тариф** | **Producer через 100 paid**, если данные подтвердят |
| **Опасные лимиты** | ① sync ingest ② daily sync × N конкурентов ③ demo 12 400 tok ④ trends poll off-tab |
| **P0 инженерия** | Async ingest queue; rate limit feed/generate; billing gates; лог `usage` DeepSeek |

**Главный вывод:** продукт **можно монетизировать** на текущих ценах, но **старые пулы токенов (4k/25k) и маржа 88% без opex — неверны**. Реальная угроза — не TikHub $/call, а **Vercel + злоупотребление Creator**.
